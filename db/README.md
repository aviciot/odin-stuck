# the-M — Database

Postgres DB: `them`, schema: `them`. Runs in `them-postgres` container (own isolated instance).

---

## Restore full dump to a new server (recommended)

`them_full_dump.sql.gz` is the canonical snapshot — schema + all data, taken 2026-07-20. Use this to clone the DB exactly.

**File location:** `db/them_full_dump.sql.gz` (relative to project root `/opt/docker/odin/`)

```bash
# 1. Start only postgres
docker compose -f docker-compose.yml -f docker-compose.local.yml up -d them-postgres

# 2. Create schemas
docker exec them-postgres psql -U them -d them \
  -c "CREATE SCHEMA IF NOT EXISTS them;" \
  -c "CREATE SCHEMA IF NOT EXISTS auth_service;"

# 3. Decompress and load
gunzip -k db/them_full_dump.sql.gz
docker cp db/them_full_dump.sql them-postgres:/tmp/them_full_dump.sql
docker exec them-postgres psql -U them -d them -f /tmp/them_full_dump.sql

# 4. Start the rest of the stack
docker compose -f docker-compose.yml -f docker-compose.local.yml up -d
```

Login: `admin` / `admin123`

## Clean install (no run history)

Schema + structural config only — no past runs, tasks, or usage data.

```bash
docker compose -f docker-compose.yml -f docker-compose.local.yml up -d them-postgres
./db/init_fresh_db.sh
docker compose -f docker-compose.yml -f docker-compose.local.yml up -d
```

## Take a new backup

Run before any major schema change or release. Produces a timestamped full dump AND refreshes all snapshot files in this directory.

```bash
./db/dump_db.sh
# then commit the updated .gz and snapshot files to git
```

## Known state as of snapshot (2026-07-20)

- **17 tables** in `them` schema, **8 tables** in `auth_service` schema
- **`orchestrators` table is intentionally empty** — all orchestration runs through `app_orchestrators` (per-application model). The table still exists because code still references it; do not drop it.
- **All agents use `a2a_async` transport** — the `agents_transport_check` constraint only allows `a2a_async`. You cannot insert an `omni_ws` agent without dropping and re-adding the constraint.
- **`app_orchestrators.edges`** — live DB type is `TEXT[]` but the SQLAlchemy model maps it as `JSONB`. Write a list of strings (`["websocket"]`), not a dict, to avoid ORM type errors.
- **Encrypted credentials in the dump** only decrypt with the `secrets.local` from the original server. After restoring on a new server, re-run `./generate-env.sh` and re-enter API keys via the admin UI.

See `docs/SCHEMA.md` for full table reference.
